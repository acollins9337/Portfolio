ROS2 Autonomous Maze Solver

The goal of our project was to develop a ROS2 software package for a TurtleBot3 that could solve a simple maze. Each wall of the maze had a sign that pointed in the direction the robot should go to solve the maze. Signs were spaced apart such that the maze could be generalized as gird made up of 3'x3' cells. We had control of the robot's initial orientation, but once the robot was placed it had to complete the maze fully autonomously. We discussed several ideas to make this work, including a bug algorithm based design, a simple "always turn right" strategy, or a ROS2 Nav2 based package.

We selected the Nav2 based package because it gave us the most versatility. We created a simple state machine to control the behavior of the robot and trained a KNN classifier for the signs. The robot defaulted to a forward motion if the path was not obstructed and only tried classification when the forward path was obstructed. If classification was successful, a Nav2 rotational waypoint would be placed in the direction of the next target cell (left, right, or backwards). Once the new heading was reached, forward motion would resume. If classification failed, fallback procedures were created. The robot could adjust its distance to the sign to attempt a better reading. If it couldn't get a better reading within a certain number of attempts, it would default to turning left (the optimal guess given the structure of the maze). A simple PID controller for the wheel output velocity was used as a low level controller.

Our final submission achieved 90% sign classification accuracy, and was able to complete the maze properly and without misclassification in 5/6 tests from multiple starting locations. Given more time, we would try to imrpove the accuracy of our KNN model under different lighting conditions, and would further try to reduce the movement error in the system.
